# Issues Log

> Track bugs, blockers, open questions, and unresolved items here.
> Update status as issues are resolved. Do not delete resolved entries.

---

## Status Definitions

| Status | Meaning |
|--------|---------|
| `OPEN` | Issue identified, not yet addressed |
| `IN_PROGRESS` | Currently being worked on |
| `BLOCKED` | Cannot proceed until dependency is resolved |
| `RESOLVED` | Fix applied and verified |
| `WONTFIX` | Deliberately not fixing (with rationale) |

## Severity Definitions

| Severity | Meaning |
|----------|---------|
| `CRITICAL` | Blocks all progress; must fix before continuing |
| `HIGH` | Blocks current block; can work on other blocks |
| `MEDIUM` | Degraded functionality; workaround exists |
| `LOW` | Cosmetic or minor; fix when convenient |

---

## Format

```
### I-{NNN}: {SHORT_TITLE}

- **Status**: {OPEN / IN_PROGRESS / BLOCKED / RESOLVED / WONTFIX}
- **Severity**: {CRITICAL / HIGH / MEDIUM / LOW}
- **Block**: {Which block is affected, or "global"}
- **Date Opened**: {YYYY-MM-DD}
- **Date Resolved**: {YYYY-MM-DD or "—"}
- **Description**: {What is the issue?}
- **Root Cause**: {Known root cause, or "unknown"}
- **Resolution**: {How it was fixed, or "pending"}
- **Notes**: {Additional context}
```
